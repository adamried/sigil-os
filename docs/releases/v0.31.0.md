# Sigil OS v0.31.0 — Release Notes

**Released:** March 19, 2026

## What's New

### Audit Mode: See What Sigil Did

You can now view a full record of what happened during any Sigil workflow. After a feature completes, run `/sigil-audit` to see which phases ran, what decisions were made, and how long each step took.

To turn audit logging on, run `/sigil-config set audit_mode true`. Once enabled, every workflow step is automatically recorded to `.sigil/audit-log.md` in your project. The log is private and stays on your machine — it's gitignored by default and won't be shared with your team.

When audit mode is active, the `/sigil` status screen shows **Audit Mode: Active | Entries: N** so you can confirm it's running at a glance.

### Setup Connects to Team Standards Earlier

If your organization uses a shared context repo, Sigil now pulls in your team's standards during setup — before your constitution is created, not after. This means team rules are woven into your project from the very first run, rather than requiring a manual sync pass to catch anything that was missed.

## What Changed

- **Shared standard template** (`templates/shared-standard-template.md`) is now included as a structured starting point for writing org-level engineering standards. Standards written with this template are automatically discovered and applied when team members run `/sigil-connect`. See `docs/shared-context-setup.md` for details. (This is primarily useful for team administrators setting up a shared context repo.)
- **Pre-execution check** now recognizes the audit log path and skips it during constitution validation.

## Upgrading from v0.30.0

No action needed. Install the update and continue using Sigil as usual. Audit mode is off by default — enable it only if you want the log.

If you manage a shared context repo for your team, consider adding a `templates/shared-standard-template.md` to give contributors a consistent structure for writing standards.

## Known Issues

- The audit log records phase names and timing but does not yet capture the full content of agent decisions. Richer log entries are planned for a future release.
